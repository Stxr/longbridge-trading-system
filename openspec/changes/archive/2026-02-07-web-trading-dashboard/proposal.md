## 原因

目前系统的所有操作都依赖 CLI（`lbt` 命令），虽然功能完备，但在以下场景下存在明显不足：
1. **可视化缺失**：回测结果仅以表格数字展示，无法直观看到 K 线走势、买卖点位置和资金曲线变化。
2. **操作门槛高**：每次回测都需要在终端中输入命令和参数，非技术用户不易上手。
3. **实盘监控不便**：CLI 无法实时展示持仓状态、盈亏变化等关键信息的图表。

需要一个 Web 版面板，将回测和实盘交易的核心功能以可视化、交互式的方式呈现。

## 变更内容

- **Web 后端 API**：基于现有模块（策略框架、回测引擎、数据管理）封装 RESTful API，供前端调用。
- **Web 前端面板**：构建单页应用，包含以下核心页面：
  - **回测面板**：选择策略、配置参数、执行回测、展示结果（K 线图叠加买卖点标记 + 资金曲线）。
  - **实盘面板**：启动/停止策略、实时展示持仓和订单状态。
  - **数据管理**：查看已同步的历史数据，触发数据同步。
- **K 线图表**：使用 `lightweight-charts` 渲染专业级 K 线图，支持买卖点标记（markers）。
- **资金曲线图**：展示回测期间的净值变化和回撤情况。

## 功能

### 新增功能

- `web-api`: 封装回测引擎、策略加载、数据管理等模块为 HTTP API 接口。可能会用到 socket，因为实盘交易数据是持续的，回测也可能是持续的。实盘交易需要从长桥读取常用数据。比如：余额，当日盈亏等。
- `web-dashboard-ui`: 前端面板，包含回测配置、K 线图表、资金曲线、实盘监控等交互界面。

### 修改的功能
CLI（`lbt` 命令）
增加 web 命令，通过 cli 一键启动网页和服务。

## 影响范围

- **新增依赖**：后端需要 `express`（或 `fastify`）；前端需要 `react`、`lightweight-charts`、`tailwindcss` 等。cli 需要增加 web 命令，用 lbt web 可以打开网页。
- **项目结构**：新增 `src/web/` 目录存放后端 API，新增 `web/` 目录存放前端应用。
- **现有模块**：不修改现有模块代码，仅通过 API 层调用已有的策略框架、回测引擎、数据管理等模块。
