## ADDED Requirements

### Requirement: 增量拉取长桥历史数据
系统应当（SHALL）能够从长桥 API 拉取缺失的历史 K 线数据并同步到本地。

#### Scenario: 自动补全最近缺失的数据
- **WHEN** 用户启动同步，本地数据最新到 2024-01-01
- **THEN** 系统向长桥请求从 2024-01-02 至今的数据并存入本地

### Requirement: 同步频率控制
同步任务必须（MUST）遵守长桥 API 的频率限制（每秒不超过 10 次请求）。

#### Scenario: 批量拉取大量数据
- **WHEN** 需要下载过去一年的分钟线
- **THEN** 系统在请求之间引入适当延迟，确保不触发 API 的频率限制错误

### Requirement: 高精度分页回溯
系统应当（SHALL）支持基于 `NaiveDatetime` 的秒级精度分页，以拉取跨度较大的分钟线数据。

#### Scenario: 同步 10,000 条分钟线
- **WHEN** 持续向前回溯历史数据
- **THEN** 系统每次利用批次中最早的时间戳作为下一批次的结束时间，确保数据无缝衔接

### Requirement: 多股票批量同步
系统应当（SHALL）支持一次性传入多个股票代码进行循环同步。

#### Scenario: 同步组合列表
- **WHEN** 传入 "700.HK, 9988.HK"
- **THEN** 系统依次为每个股票执行同步逻辑
